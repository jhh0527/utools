2. 입력 형식
--------------------------------------------------------------------------------
- 사용자는 SRT 자막 파일을 제공한다.
- SRT 파일은 번호, 타임코드, 자막 텍스트로 구성된다.
- 썸네일 생성 개수는 **요청자가 명시**한다. (미명시 시 썸네일 0개 — 본문 씬만 생성)

SRT 형식 예시:
  1
  00:00:00,000 --> 00:00:03,500
  안녕하세요, 오늘은 왜 사람은 잠을 자야 하는지 풀어볼게요.

  2
  00:00:03,500 --> 00:00:07,200
  수면 중에 뇌는 하루 동안 쌓인 노폐물을 정리한다는 연구가 있습니다.


3. 이미지 수 산정
--------------------------------------------------------------------------------

3-1. 총 영상 길이 계산
- SRT의 마지막 자막 종료 시간을 초 단위로 변환
- 예: 00:18:17,068 → 18×60 + 17 + 0.068 = 1097초

3-2. 타임스탬프 생성 규칙 (매칭 기준점 전용 — 파일명/라벨로 사용 금지)

【구간 1: 0~180초 (3분까지, 필수 고정)】
- 15초 간격으로 13개 고정 생성 (무조건)
- 타임스탬프: 0, 15, 30, 45, 60, 75, 90, 105, 120, 135, 150, 165, 180

【구간 2: 200초 이후】
- ★★★ 20초 간격으로 생성 (절대 규칙) ★★★
- 시작: 200초
- 종료: (마지막 SRT 번호 - 5)번 항목의 시작시간 이하까지

★★★ 중요: 위 타임스탬프는 SRT 매칭을 위한 "기준점"일 뿐이며,
출력 시 파일명·라벨에는 절대 사용하지 않는다. 라벨은 항상 매칭된 SRT의 시작초이다. ★★★

3-3. 마지막 이미지 제한 규칙 (필수)
- 마지막 프롬프트에 매칭되는 SRT 항목은 (전체 SRT 항목 수 - 5) 이하여야 한다.

3-4. 총 필요 이미지 수
- 공식: 13 + 구간 2 타임스탬프 개수
- 구간 2 개수 = floor((종료한도 - 200) / 20) + 1  (종료한도 ≥ 200일 때)


4. 타임스탬프 → SRT 씬 매칭 알고리즘
--------------------------------------------------------------------------------

4-1. 기본 규칙
- 1번 SRT 항목부터 매칭 시작 (모든 SRT 항목이 매칭 대상)
- 각 타임스탬프에 1:1로 SRT 항목을 매칭 (중복 선택 불가)

4-2. 매칭 알고리즘
각 타임스탬프 T에 대해:
  단계 1: T 이하 시작시간 중 아직 선택되지 않은 항목 가운데 시작시간이 가장 큰 항목 선택
  단계 2: 없으면 T 초과 시작시간 중 아직 선택되지 않은 항목 중 시작시간이 가장 작은 항목 선택

4-3. 매칭 제한
- 매칭된 SRT 번호가 (전체 SRT 항목 수 - 5)를 초과하면 해당 타임스탬프 프롬프트 생성 안 함

4-4. ★★★ 매칭 결과 → 파일명 변환 규칙 ★★★
- 매칭된 SRT 항목의 **시작 시간(초, floor)** 을 S로 추출
- 파일명 = `SRT_{S, 최소 3자리 zero-padding}.png`
  · 00:00:03,500 → `SRT_003.png`
  · 00:02:30,000 → `SRT_150.png`
  · 00:18:09,200 → `SRT_1089.png`
- 1000초 이상은 자릿수 그대로 사용 (4자리 이상 허용)
- floor 충돌 시 뒤 항목은 +1초 조정
- ★ 라벨 = 매칭 SRT의 실제 시작초 → 대본과 1:1 일치 ★


5. SRT 내용 기반 장면 판별
--------------------------------------------------------------------------------

5-1. ★★★ 핵심 요소 추출 (프롬프트 작성 전 필수) ★★★

[행동]·[설정]·[조연] 자리에 **시각적 형상으로만** 반영한다. (이미지 내 글자 금지)

  ① 핵심 명사(주체/객체)
  ② 핵심 동사(행동/변화)
  ③ 숫자/연도/단위 → ★ 글자로 표시하지 말고, 시각적 크기·개수·도표 형상·스케일 대비로 표현 ★
  ④ 고유명사(인물/장소/기관) → 상징적 외형/유물/소품으로 표현 (이름 글자 금지)
     · 사람일 경우 Whatly Dreamer + 상징적 소품(왕관·청진기·돋보기 등 단순 플랫 실루엣)으로만 표현
  ⑤ 비교/대립 구조: A vs B → 좌우 분할 구도
  ⑥ 인과관계: 원인 → 결과 시각화
  ⑦ 감정 톤: 호기심/긍정/부정/경고/반전/대결

5-4. ★★★ SRT 내용 → 시각 요소 변환 규칙 (필수) ★★★

  ★ 글자 표시 금지 원칙 ★
  - 숫자/연도/통계는 모두 **시각적 형상**으로 변환한다:
    · "8시간" → 시계가 8 구획으로 시각 분할된 플랫 형상, 또는 달·베개 8개 패턴
    · "476년" → 부서진 로마 양식 건축물 플랫 실루엣 + 무너지는 깃발
    · "70%" → 화면의 약 70%를 채우는 색·차오른 액체·차트 막대 길이 (숫자 글자는 그리지 않음)
    · "10배 차이" → 한쪽이 다른 쪽보다 명백히 10배 크게 보이는 스케일 대비
    · "두 학파 대립" → 좌우 분할, 두 Whatly Dreamer·유물의 대치 구도
  - 일반론 금지: "지식을 설명하는 호스트"가 아니라, 그 자막이 말하는 **바로 그 대상**을 그린다.

    → human_required 판정 → human_required=true일 때만 Whatly Dreamer 추가
    → SRT 주제에 맞는 whatly_scene_variant 선택 (§8-3)

14-2. ★ 프롬프트 작성 후 추가 체크 ★
  □ 프롬프트에 `Korean text "..."` 또는 displayed text 표현이 남아 있지 않은가?
  □ 소품(책·배너·간판·달력·차트·시계·지도·종이·동전·지폐)에 글자가 그려지지 않는가?
  □ 숫자·연도·통계가 시각적 형상으로 변환되어 표현되었는가?
  □ 글자 금지 키워드(no text, no numbers, no letters, no captions, no labels,
     no signage text, no written words on objects)가 포함되어 있는가?
  □ 실사 금지 키워드(no photorealistic, no photography, no DSLR, no live-action)가 포함되어 있는가?
  □ Whatly 스타일 키워드(minimalist 2D flat illustration, Whatly Dreamer,
     round circle head, messy tousled hair, expressive eyes nose mouth,
     rich facial expression, thin stick-like legs, simple flat torso,
     flat color fill, no outlines, clean vector art)가 포함되어 있는가?
  □ 금지 스타일 키워드(no anime, no cartoon, no manga, no Studio Ghibli,
     no 3D render, no anthropomorphic mascots)가 포함되어 있는가?
  □ 실사·일반인 체형 금지 키워드(no photorealistic human, no hyper-realistic portrait,
     no normal adult human body proportions, no realistic muscular legs)가 포함되어 있는가?
  □ 사람이 등장한다면 Whatly Dreamer(둥근 얼굴·더벅머리·풍부한 표정·가는 다리) 표현이 명시되어 있는가?
  □ 일반인 체형·dot eyes only·no nose·bald·무표정 얼굴이 남아 있지 않은가?
  □ Whatly 색상 팔레트(§8-2)가 배경·캐릭터에 반영되어 있는가?
  □ 라벨이 매칭 SRT 시작초와 일치하는 `SRT_XXX.png` 형식인가?
  □ 말풍선 트리거 없음
  □ Scene Blueprint forbidden_elements에 텍스트·실사·지브리 금지가 들어있는가?
  □ SRT 일치도 80점 이상인가?

14-3. 최종 검증
  □ 모든 이미지에 글자(한글/영문/숫자/기호)가 표현되지 않도록 명시되었는가?
  □ 모든 배경이 Whatly 2D 플랫 일러스트 스타일로 명시되었는가?
  □ 모든 사람이 Whatly Dreamer(둥근 얼굴·더벅머리·풍부한 표정·가는 다리)로만 표현되도록 명시되었는가?
  □ 모든 프롬프트에 §6 필수 스타일 문구가 포함되었는가?

※ 본 섹션은 내부 처리용이며 최종 출력에 절대 포함하지 않는다.
